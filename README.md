# BatteryAI

**Платформа прогноза и диагностики аккумуляторных батарей электромобилей**

> Дипломный проект · React + TypeScript · FastAPI · MongoDB · Railway

---

## О проекте

**BatteryAI** — веб-платформа для мониторинга, диагностики и прогнозирования состояния аккумуляторных батарей электромобилей. Работает в браузере на любом устройстве. Данные обновляются в режиме реального времени каждые 5 секунд.

### Что умеет

- Показывает здоровье батареи (SoH), уровень заряда (SoC), температуру и остаточный ресурс (RUL)
- Строит прогноз деградации на 24 месяца вперёд
- Генерирует предупреждения при критических значениях
- Сравнивает батареи двух автомобилей по 5 метрикам
- Показывает историю зарядок и экспортирует отчёт в CSV
- Симулирует подключение по USB/OBD-II для добавления авто

## Запуск локально

### Требования

- Node.js 18+
- Python 3.10+
- MongoDB (локально или Atlas)

### Frontend

```bash
cd frontend
npm install
npm run dev
```

Откроется на `http://localhost:5173`

### Backend

```bash
cd backend
pip install -r requirements.txt
uvicorn app.main:app --reload --port 8080
```

API доступно на `http://localhost:8080`

### Переменные окружения

Создайте файл `backend/.env`:

```env
MONGODB_URL=mongodb://localhost:27017
SECRET_KEY=your_secret_key
DATABASE_NAME=evbis
```

## Страницы

### Главная (`/`)
Обзор всех автомобилей. Агрегированные метрики флота, карточки с круговым индикатором SoH, живые спарклайны.

### Мои автомобили (`/vehicles`)
Список ТС. Добавление вручную или через симуляцию USB/OBD-II сканирования (7 шагов: CAN → VIN → BMS → параметры).

### Детали авто (`/vehicles/:id`)
Полный анализ батареи: SoC-gauge, прогноз деградации, ток/температура, балансировка ячеек, история зарядок, экспорт CSV.

### Сигналы (`/alerts`)
Предупреждения 4 уровней (Критический / Высокий / Средний / Низкий). Новые алерты каждые 25–45 сек, счётчик в сайдбаре.

### Аналитика (`/analytics`)
Donut-диаграмма состояния батарей, рейтинг по SoH, прогноз деградации с выбором авто.

### Сравнение (`/compare`)
Выбор двух авто → анимированный 8-шаговый анализ → победитель по 5 метрикам, радарная диаграмма.

## Ключевые показатели

| Показатель | Расшифровка | Норма |
|-----------|-------------|-------|
| **SoH** | State of Health — здоровье батареи | > 80% |
| **SoC** | State of Charge — уровень заряда | > 20% |
| **RUL** | Remaining Useful Life — остаток ресурса | > 300 циклов |
| **EOL** | End of Life — конец ресурса | 80% SoH |
| **IR** | Internal Resistance — внутр. сопротивление | < 16 мОм |

## Деплой на Railway

Проект состоит из двух сервисов:

**Frontend** — собирается через Dockerfile, nginx отдаёт статику и проксирует `/api/*` на backend:
```nginx
proxy_pass http://backend.railway.internal:8080;
```

**Backend** — FastAPI в Docker-контейнере на порту `$PORT`.

При каждом `git push origin main` Railway автоматически пересобирает оба сервиса.
